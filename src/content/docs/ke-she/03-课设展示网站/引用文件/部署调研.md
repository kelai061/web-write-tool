## 1. 背景与选型关键点

### 1.1 项目情况

`tool\` 跑完 `npm run build` 后产出 `dist\` **纯静态 HTML 文件**（无数据库、无后端），所以**任何静态托管平台都能用**。

### 1.2 选型只看四件事

| 你的诉求    | 影响                   |
| ------- | -------------------- |
| 想免费     | 排除国内云服务器（要花钱买服务器）    |
| 不想折腾命令行 | 优先选"连 GitHub 一键部署"的  |
| 想国内访问快  | 要么走海外 CDN（够用），要么必须备案 |
| 希望长期稳定  | 走大平台，别走小众服务          |

## 2. 方案 A：Vercel + Cloudflare 中转

> 这是 `项目档案与备份.md` 7.4 节的**原方案**，下面补充它的国内访问问题。

- **怎么部署**：GitHub 推代码 → Vercel 自动构建 → 拿到 `xxx.vercel.app`
- **国内访问**：Vercel 直连国内**经常抽风**（被墙、加载慢、有时打不开）。解决办法是把**自定义域名**接到 Cloudflare，走 Cloudflare 的边缘 CDN 中转，国内访问会明显变稳。
- **备案**：不需要
- **费用**：全免费
- **操作难度**：⭐⭐⭐（要会改 DNS、配 Cloudflare）
- **缺点**：双平台跳来跳去，新人容易迷糊；Vercel 有时会封号（建站太多）

---

## 3. 方案 B：Cloudflare Pages【推荐】

- **怎么部署**：连 GitHub 仓库 → 选 Astro → 自动构建 → 拿到 `xxx.pages.dev`
- **国内访问**：比 Vercel 稳定得多，Cloudflare 国内能直连，延迟可接受（200ms 左右）
- **备案**：不需要
- **费用**：全免费，**额度最大方**（无限请求、100GB/月构建）
- **操作难度**：⭐⭐（图形界面点点点，比 Vercel 还简单）
- **优点**：CDN 全球节点多、对 Astro 识别好、绑定自定义域名极简单
- **缺点**：偶尔部分地区会慢；自定义域名才能享受完整 CDN 加速

---

## 4. 方案 C：GitHub Pages

- **怎么部署**：GitHub 仓库 → Settings → Pages → 选分支 → 自动出 `username.github.io/仓库名/`
- **国内访问**：能访问，**但稳定地慢**（100ms+，偶尔抽风）
- **备案**：不需要
- **费用**：完全免费
- **操作难度**：⭐（最简单，纯 GitHub 里点点点）
- **优点**：和你写代码的 GitHub 同一家，零额外账号
- **缺点**：国内速度一般；**百度不收录**（SEO 短板）

---

## 5. 方案 D：腾讯云 EdgeOne Pages

- **怎么部署**：连 GitHub → 自动构建 → 拿到 `xxx.edgeone.app`
- **关键区分（重要）**：

| 维度     | 国际站（pages.edgeone.ai） | 国内站（cloud.tencent.com）   |
| ------ | --------------------- | ------------------------ |
| 自定义域名  | **无需备案**              | **必须完成国内 ICP 备案**        |
| CDN 加速 | 海外 CDN 节点（国内能访问）      | 国内 + 全球 CDN 节点（**速度最快**） |
| 域名验证   | 自动                    | 需手动点击验证                  |

- **费用**：基础免费（注意注册时可能显示"试用 14 天"，要看清套餐）
- **操作难度**：⭐⭐（图形界面）
- **适合**：如果你**愿意花 1-2 周走备案流程**（需要买国内域名 + 个人备案），这是国内访问体验最好的方案
- **缺点**：备案麻烦，对个人小项目**性价比不高**

---

## 6. 方案 E：国内云厂商 OSS + CDN

- **怎么部署**：`dist\` 上传到 OSS 存储桶（阿里云/腾讯云）→ 开 CDN → 绑域名
- **国内访问**：最快最稳
- **备案**：**必须备案**（绑国内域名 + CDN 都要求）
- **费用**：OSS 几毛/月，CDN 按流量（个人几十块/年）
- **操作难度**：⭐⭐⭐⭐（要配置的东西多）
- **适合**：访问量大、要做正经业务的网站。**对个人文档站，杀鸡用牛刀**

## 8. 最终推荐

针对本项目（**个人文档站、想免费、想国内能访问、不想折腾**）的需求：

### 第一选择：Cloudflare Pages

理由：

1. **国内访问比 Vercel 稳定**（最头疼的国内访问问题直接解决）
2. **不用备案**（省 2 周时间）
3. **免费额度最大**（不担心超量）
4. **图形界面操作**，连 GitHub 仓库 → 选 Astro → Deploy，3 步搞定
5. 和档案 7.4 节"Vercel 部署"的流程几乎一样，只是换个网站

### 备选：GitHub Pages

如果连 Cloudflare 都嫌麻烦，直接用 GitHub Pages，**就在写代码的 GitHub 里点几下**就能上线，国内虽然慢但能访问，足够个人文档站用。

### 何时升级到国内 CDN 方案

如果后续发现国内速度不够、且愿意花时间备案，再考虑迁到 **EdgeOne 国内站** 或 **OSS + CDN**。

## 9. Cloudflare Pages 部署操作流程

> 以下提示词可直接交给 AI 执行，或自己照着做。

```
帮我把 E:\AI\zcode\object\Web imitation\tool\ 部署到 Cloudflare Pages。

步骤：
1. 初始化 git 并推到 GitHub：
   cd tool
   git init
   git add .
   git commit -m "initial"
   （先在 github.com 建一个 public 仓库）
   git remote add origin https://github.com/【用户名】/【仓库名】.git
   git branch -M main
   git push -u origin main

2. 去 https://dash.cloudflare.com 注册/登录（GitHub 一键登录也行）

3. 左侧菜单 Workers & Pages → Create application → Pages → Connect to Git

4. 选刚才的仓库，配置：
   - Framework preset: Astro
   - Build command: npm run build
   - Build output directory: dist
   - 环境变量 NODE_VERSION = 24

5. Save and Deploy → 等 1-2 分钟，拿到 https://【仓库名】.pages.dev

6. （可选）自定义域名：Pages 项目 → Custom domains → 添加域名

注意：
- .gitignore 已排除 node_modules\ 和 dist\
- 不需要备案
- 以后 git push 自动重新构建发布
```
