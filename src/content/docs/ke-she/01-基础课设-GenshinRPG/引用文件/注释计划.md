---

## 0. 背景与现状

上一轮重构已经把项目分包、拆类、加了方法级 Javadoc。但**按钮→运转**的可读性还不够：
- 按钮上方的注释只写了"这一行在干嘛"，没写"它会跳到哪些方法、数据流向哪里"。
- 方法级 Javadoc 写了"作用/输入/输出"，但**没写"我是被谁调用的"**，读者跳进去之后不知道自己从哪来。
- 跨文件调用（Panel → MainFrame → CombatSystem/StorageManager）缺少"路标"。

本计划要做的事：**给每个按钮加一张 ASCII 调用链路图，给每个被按钮（间接）调用的方法加一行【被谁调用】标记**。

---

## 1. 唯一原则

**只加注释，不改代码。**
- 不许改任何一行可执行代码（包括空格、换行位置）。
- 不许改 import、package、方法签名、字段。
- 不许"顺手优化"或重命名。
- 注释一律用中文，编码 UTF-8。
- 改完必须能 `javac -encoding UTF-8` 零 error 编译通过。

## 2. 注释模板规范

### 模板 A：按钮上方的【调用链路图】（用在每个 `addActionListener` 之上）

格式固定为带框线的块，箭头 `→` 表示调用顺序，括号内注明所在类：

```
// ╔═══════ 按钮：<按钮中文名> ═══════╗
// 链路：
//   本按钮(<当前类>)
//     → <方法名>(<类>)           // 一句话说明这一步做什么
//     → <方法名>(<类>)           // 一句话
//     → ...直到副作用落地（写库/扣血/切页/弹窗）
// 一句话总览：<点下去最终发生了什么>
// ╚══════════════════════════════════╝
btnXxx.addActionListener(e -> ...);
```

要求：
- 链路图必须**完整到副作用落地**为止（写库/改血量/切页/弹提示框），不能停在中间方法。
- 跨类调用要标明类名，例如 `→ playerAction(CombatSystem)`。
- 每行用 `// 一句话` 说明该步在做什么，让读者不用跳进去也大致明白。
- 若按钮逻辑极简（如直接 `parent.showPanel("menu")`），链路图可缩为两三行，但必须有框线头尾。

### 模板 B：方法顶部的【被谁调用】标记（加在已有 Javadoc 之后、方法体之前）

凡是被按钮**直接或间接**调用的方法，在 Javadoc `*/` 之后、方法签名之前，插入一行：

```
     */
    // 【被谁调用】来自 <调用方类>.<调用方方法>（按钮：<按钮名>）；本方法还会调用：<下游方法(类)>, ...
    public/private ... methodName(...) {
```

若一个方法被多处调用，用顿号或"或"列全；若无任何调用方（入口/回调），写 `// 【被谁调用】入口方法 / 由 Swing 回调触发`。

### 模板 C：单行就地补充（可选，少用）

在方法体内某个关键步骤上方，若该步骤是链路图里点名的"落地动作"（如真正扣血、真正写库、真正切页），加一行 `// ▶ 落地：...`。**每条链路最多标 1~2 个落地点**，避免泛滥。

## 4. 跨层"枢纽方法"必须加【被谁调用】（执行 AI 重点处理）

这些方法是被多个按钮共用、读者最容易迷路的"枢纽"，必须补【被谁调用】+【下游】：

### MainFrame（app/MainFrame.java）
- `showPanel(String)` ——【被谁调用】几乎所有 Panel 的按钮/返回；本方法先 `SM.saveUserProgress`(自动存档)，再调目标 Panel 的 `refresh()`，再 `cardLayout.show`。
- `loginAsUser(User)` ——【被谁调用】LoginPanel.handleSubmit(玩家登录按钮)；调 `SM.loadUserProgress` 后 `showPanel("menu")`。
- `logout()` ——【被谁调用】MainMenuPanel.btnExit；先存档再 `initGameData` 重置，再 `showPanel("login")`。
- `startLevelChallenge(int)` ——【被谁调用】LevelPanel.startChallenge(关卡按钮)；设关卡 id/波次后调 `startWave`。
- `startWave()` ——【被谁调用】MainFrame.startLevelChallenge 与 handleBattleEnd(下一波)；`SM.loadEnemies` 生怪→`new CombatSystem().startBattle`→`BattlePanel.initBattle`→`showPanel("battle")`。
- `createScaledEnemy(Stats)` ——【被谁调用】startWave（按世界等级缩放怪物三围）。
- `handleBattleEnd(boolean)` ——【被谁调用】BattlePanel.checkBattleEnd；发奖励/回血/判断下一波/世界等级提升/切回主菜单。
- `initGameData()` ——【被谁调用】构造函数 与 logout；`SM.initTables`+`loadCharacters`+`SkillSystem.loadSkillsForCharacter`+默认编队+`inventory.initInventory`。

### CombatSystem（service/CombatSystem.java）
- `startBattle(...)` ——【被谁调用】MainFrame.startWave；初始化队伍/敌人/护盾。
- `playerAction(int, Enemy)` ——【被谁调用】BattlePanel.handleAttackAction(攻/技/大招按钮)、handleSwapAction(切人)、handleUseItemAction/handle内联(用食物)；是"我方一回合"总入口，内部：`executeTurn`→敌方 `takeTurn`→`updateStates`。
- `executeTurn(...)` ——【被谁调用】playerAction；执行单次技能：技能.execute→triggerReaction→calculateFinalDamage→takeDamageWithLog→反应后续效果。
- `triggerReaction(...)` ——【被谁调用】executeTurn；判定元素反应类型并算增幅/固伤加成。
- `calculateFinalDamage(...)` ——【被谁调用】executeTurn；算最终伤害（暴击/防御/抗性/反应）。
- `applyTransformativeReaction(...)` ——【被谁调用】executeTurn(超载/超导/感电/扩散/碎冰/燃烧/烈绽放/超绽放)；剧变反应伤害。
- `isTeamDefeated()` / `isAllEnemiesDefeated()` ——【被谁调用】BattlePanel.checkBattleEnd。
- `saveLogToFile()` ——【被谁调用】MainFrame.handleBattleEnd；导出 txt 战报 + `SM.saveBattleLog`。

### 各 Panel 的私有 handler（务必加【被谁调用】）
LoginPanel.handleSubmit/updateUIForMode；LevelPanel.startChallenge；TeamPanel.applyTeamChanges；UpgradePanel.useExpBook/buyExpBook/updateDetails/onCharacterSelected；BagPanel.useItem/updateDetails/onItemSelected；AdminPanel.refreshUserList/triggerAddUserDialog/triggerDeleteUser/triggerSaveChanges/loadSelectedUserData/clearRightPanel；BattlePanel.handleAttackAction/handleSwapAction/handleUseItemAction/showTargetCharacterMenuForFood/checkBattleEnd/refreshGUI/initBattle。

### StorageManager 关键 API（dao/StorageManager.java，每个都加【被谁调用】）
- `initTables` ← MainFrame.initGameData
- `loadCharacters`/`loadEnemies`/`loadSkills` ← MainFrame.initGameData / startWave / SkillSystem.loadSkillsForCharacter
- `loginUser`/`registerUser` ← LoginPanel.handleSubmit
- `loadUserProgress` ← MainFrame.loginAsUser
- `saveUserProgress` ← MainFrame.showPanel(自动存档)/logout；UpgradePanel/BagPanel/BattlePanel 用物后
- `getAllUsers`/`addUser`/`updateUser`/`deleteUser` ← AdminPanel 各按钮
- `getUserCharacters`/`updateUserCharacter`/`getUserItems`/`updateUserItem` ← AdminPanel.loadSelectedUserData/triggerSaveChanges
- `saveBattleLog` ← CombatSystem.saveLogToFile

### Inventory.useItem（model/Inventory.java）
- 【被谁调用】UpgradePanel.useExpBook(itemId=4)；BagPanel.useItem(id=1/2/3)；BattlePanel 战斗用食物(id=1/2/3)。

### Skill 各子类 execute（service/skill/*.java）
- 【被谁调用】CombatSystem.executeTurn；返回技能倍率并附元素/附魔。

## 5. 工作量与做法

1. **按界面顺序**处理：Login → MainMenu → Level → Team → Upgrade → Bag → Admin → Battle。
2. 每个界面：先在**每个按钮的 `addActionListener` 上方**插入【链路图】（模板 A），内容照抄第 3 节对应条目。
3. 再在该界面**每个 handler 私有方法**顶部加【被谁调用】（模板 B）。
4. 处理 MainFrame、CombatSystem、StorageManager、Inventory、Skill 子类的枢纽方法，补【被谁调用】。
5. **不动任何代码行**。只加注释。
6. 全部完成后，清空 `bin\`，运行 `run.bat` 重新编译，确认 0 error。

## 7. 风险提示

- 链路图里的类名/方法名必须**与现有代码一字不差**（如 `handleAttackAction` 不能写成 `handleAttack`）。
- 若某方法被多个按钮调用，【被谁调用】要**列全**，不要漏。
- 不要把链路图画得太长导致换行错乱；每行控制在 80 字符内，超长可折行用 `//     → ...` 续行。
- UTF-8 编码务必保持，中文/emoji 不能乱码。
