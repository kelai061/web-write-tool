## 正文（以下整段复制 ↓↓↓）

你是一名 Java 工程师，现在要给一个已经重构好的学生课设项目做**纯注释增强**，让初学者从任意按钮出发，顺着注释就能一路看懂程序怎么运转。

### 工作目录与必读
- 目标源码根：`E:\AI\zcode\object\ke-she\refactored\src\com\genshin\rpg\`
- 必读：先用工具读取该目录下全部 `.java` 文件，以及同目录上级的《链路注释计划书.md》（含完整按钮链路清单与模板）。
- 项目包结构：`com.genshin.rpg.{app, view, view.ui, model, model.enums, service, service.skill, dao}`。

### 铁律（违反即失败）
1. **只加注释，不改任何代码**。不许动可执行代码行、import、package、方法签名、字段、空格、数值、UI、文案。改动前后用 diff 自查：除注释行外必须零差异。
2. 注释一律**中文、UTF-8**，不能出现乱码。
3. 不许顺手优化、重命名、合并方法。
4. 全程不动 `E:\C\cDownKeShe\`（那是原版，只读参考）。

### 任务
给项目补两类注释，让"按钮→运转"可读：

**模板 A：每个按钮 `addActionListener` 上方，加一张带框线的调用链路图。** 格式：
```
// ╔═══════ 按钮：<中文名> ═══════╗
// 链路：
//   本按钮(<当前类>)
//     → <方法名>(<类>)           // 这一步做什么
//     → <方法名>(<类>)           // 这一步做什么
//     → ...直到副作用落地（写库/扣血/切页/弹窗）
// 一句话总览：<最终发生什么>
// ╚══════════════════════════════════╝
```
要求：链路必须**画到副作用落地为止**；跨类调用要标类名（缩写：MainFrame=MF, CombatSystem=CS, StorageManager=SM, Inventory=Inv, SkillSystem=SS）；每行 ≤80 字符，超长折行续写。

**模板 B：被按钮直接或间接调用的方法，在 Javadoc `*/` 之后、方法签名之前，加一行【被谁调用】。** 格式：
```
     */
    // 【被谁调用】来自 <调用方类>.<调用方方法>（按钮：<按钮名>）；本方法下游调用：<下游(类)>, ...
    public/private ... methodName(...) {
```
被多处调用要列全；入口/Swing 回调写 `// 【被谁调用】入口方法 / 由 Swing 回调触发`。

（可选、少用）模板 C：方法体内关键落地动作上加 `// ▶ 落地：...`，每条链路最多 1~2 处。

### 全量按钮链路（照此画图，类名/方法名必须与代码一字不差）

- btnSubmit 玩家注册：`本按钮 → handleSubmit(LoginPanel) → SM.registerUser(SM,写 users+初始化角色/物品) → 切回登录模式`。总览：建号并生成默认角色与背包。
- btnSubmit 管理员：`本按钮 → handleSubmit(LoginPanel) → 比对硬编码密码 "klee is comming" → MF.showPanel("admin")(MF)`。总览：密码对则进后台。

**MainMenuPanel**
- btnChallenge：`本按钮 → MF.showPanel("level")(MF,自动存档) → LevelPanel.refresh`。
- btnTeam：`本按钮 → MF.showPanel("team")(MF,自动存档) → TeamPanel.refresh`。
- btnUpgrade：`本按钮 → MF.showPanel("upgrade")(MF,自动存档) → UpgradePanel.refresh`。

**UpgradePanel**

**BagPanel**
- btnUse：`本按钮 → useItem(BagPanel) → Inv.useItem(itemId,目标,队伍,inBattle=false)(Inv,回血/复活/加buff) → SM.saveUserProgress(SM) → refresh`。
