# 《原神》文字战斗模拟系统 —— 重构计划（供 AI 执行阅读）

> 源项目根目录：`E:\C\cDownKeShe`（默认包，`javac src/*.java` 编译，`java -cp bin;mysql-connector-j-8.0.33.jar Main` 运行）。

## 0. 一句话目标

把"23 个平铺在 `src/` 的 `.java` 文件、默认包、单文件多类、800 行大文件"重组成**按 Java 规范分包 + 单文件单类 + 每个按钮事件就地写清"调用什么/做什么" + 每个方法补注释**的版本，保证**逐字节等价的功能与数值**，让初学者能顺着读。

## 1. 不可变更的"红线"（违反即失败）

重构**禁止改动**以下内容（否则功能/表现会变）：

1. **包结构是否可用默认包**：原项目是默认包（无 `package` 声明）。重构后**改为显式分包** `com.genshin.xxx`，需要同步修改 `run.bat` 与 `.iml`，并验证能编译运行（详见第 8 节）。
2. **`ElementType` 枚举顺序**：DB 用 `ElementType.values()[element列]` 还原，顺序必须保持
   `ANEMO, GEO, HYDRO, PYRO, CRYO, ELECTRO, DENDRO, PHYSICAL`（索引 0..7）。**一个都不能换**。
3. **`ReactionType` 枚举常量名与顺序**。
4. **所有数值公式**：`calcDefMultiplier`、`calcResMultiplier`、`getLevelBaseDmg`、`calcEmBonus`、世界等级缩放（`(worldLevel-1)*0.6/0.4/0.3`）、奖励金币 `50*wl*85`、经验 `100*wl*85`、奖励掉落（鸡×2 蛋×1 书×2）、战斗失败回血 `HP/能量 → 50%`、仙跳墙 `ATK+20% 暴击+12% 3回合`、经验书 `+5000EXP`、买书 `-200金币`、升级 HP/ATK/DEF 增量 `50*i / 5*i / 2.5*i`、深渊法师/岩盔王护盾数值与减伤比例、狂暴激怒 `HP<30% ATK*1.5`、超导 `-40%物抗 8回合`、护盾吸收倍率 `2.5/1.0` 等。**所有数字、系数、阈值原样搬运**。
5. **技能特例判定**（靠角色 id 硬编码）：
   - `id==1 可莉` / `id==3 莫娜` / `id==6 甘雨` / `id==8 纳西妲` → 普攻带元素
   - `id==7 刻晴` E 技能给雷附魔 2 回合
   - `id==5 迪卢克` 大招给火附魔 3 回合
   - `id==8 纳西妲` 大招 +250 精通 3 回合
   - `id==4 琴` 且 `skill.id==12` 大招按 `ATK*1.2+3000` 全队回血
   - 怪物 `specialType` 1..6 的 AI 行为（见 `Enemy.takeTurn`）
6. **关卡-怪物配置**（`MainFrame.startWave`）：1~5 关、第 2/3/4 关分波次、深渊法师按出现顺序命名为火/水/冰法师，第 5 关 3 个法师。**配置顺序与索引不动**。
7. **管理员后台密码**：`"klee is comming"`（提示语 `klee is c...`）。
8. **MySQL 自动启动路径**：`E:\SQL\mysql\bin\mysqld.exe`、`E:\SQL\mysql\my.ini`、工作目录 `E:\SQL`、`Thread.sleep(4000)`、密码候选 `["", "root", "123456"]`、连接串 `jdbc:mysql://localhost:3306/...`、库名 `genshin_rpg`、表结构与 seed 数据。**离线 fallback 的静态数据也要原样保留**。
9. **UI 视觉**：所有颜色（如 `new Color(30,30,30)` 暗底、`255,190,40` 金、`47,85,151` 蓝）、字体（`Microsoft YaHei`）、尺寸、布局、中文文案、JOptionPane 提示框文字。**一字不改**。
10. **行为流程**：登录自动保存、切页 `showPanel` 自动 `saveUserProgress` 并 `refresh`、战斗内 `useItem` 同步进度、注销时 `initGameData()` 重置、战报存文件 `battle_log_level_X_<时间戳>.txt` 等。
11. **存档键约束**：`user_characters`/`user_items` 的 PK 与外键、`ON DUPLICATE KEY UPDATE` 写法。

> ⚠️ 任何"顺手优化"（如把 for 改 stream、把魔法数字提成常量、改 buff 叠加逻辑、合并重复 switch）**一律不做**。本任务只搬不改。

## 5. 注释规范（每个方法必须补）

每个方法（含构造、私有、匿名内部类方法除外但鼓励）补一段 Javadoc 或行注释，至少包含：

- **作用**：一句话说清做什么。
- **输入**：参数含义（无参写"无"）。
- **输出/副作用**：返回值含义，或"修改了哪些字段/写库/改 UI"。

示例（保持原数值，只加注释）：

```java
/**
 * 计算防御力减伤系数。
 * 作用：根据攻击者与目标等级、目标防御减免/无视，得出 0~1 的乘数。
 * 输入：charLv 攻击者等级；enemyLv 目标等级；defReduction 目标防御减免；
 *       defIgnore 防御无视比例。
 * 输出：防御乘数（数值越小，目标越硬）。
 */
public double calcDefMultiplier(int charLv, int enemyLv, double defReduction, double defIgnore) { ... }
```

按钮事件**就地注释**："点击后：① 调 xxx ② 改 yyy ③ 刷新 UI"，紧贴在 `addActionListener` 之上。

## 6. 逐文件改造要点

### 6.1 Main.java

- 加 `package com.genshin.rpg;`，`import com.genshin.rpg.app.MainFrame;`。
- 主体逻辑（系统外观、`invokeLater`）一行不改。

### 6.2 MainFrame.java（app/）

- package + import 调整。
- **推荐**：保留全部状态字段与方法（简化方案，不引 GameContext）。
- 给 `initGameData / showPanel / loginAsUser / logout / startLevelChallenge / startWave / createScaledEnemy / handleBattleEnd` 逐个补注释，标明"输入/输出/副作用（是否写库、是否切页）"。
- 数值（`50*worldLevel*85` 等）、关卡-怪物索引、奖励掉落 id 映射**原样**。

### 6.3 八个 Panel（view/）

- 每个 Panel 构造里 `parent` 字段保留（指向 MainFrame），便于就地调用，符合"不要跳转"。
- 按钮事件**集中在本文件内**：把原本散在 `triggerXxx`/`handleXxx` 的私有方法就近写在事件里或紧邻方法，并加"①②③"注释。**不强求把多行逻辑塞进 lambda**——可保留独立方法，但要在方法体顶部用注释串起来，让读者一眼看清调用链。
- 删除 Panel 内重复的元素 `switch`（`getElementChinese`/`getElementColorHex`），统一调 `view.ui.ElementStyle`。BattlePanel 里的 `getElementColor`(返回 Color) 同样收口。
- AdminPanel 的 `AdminCharData/AdminItemData` 引用改为 `StorageManager.AdminCharData`（若拆出 UserData 则改对应类型）——**保持名字不变**，减少改动面。

### 6.4 模型层（model/）

- `ElementAttach` 从 Character.java 顶部抽成独立 public 类（原本是包级 `class ElementAttach`，要改成 `public`，因为跨包）。
- 其余原样加 package。

### 6.5 服务层（service/）

- `SkillSystem` 独立成 `public class`，原为包级；提供 `getInstance/loadSkillsForCharacter/getSkillsForCharacter/resetCooldowns`，签名不变。
- `CombatSystem` 原样搬运，补注释。`getHTMLColorString/getElementChineseName/getReactionChineseName` 可改为调 `ElementStyle`（行为等价）。

### 6.6 DAO 层（dao/）

- `DBConnector`：`connectDB/tryConnect`（含自动启动 MySQL 的全部副作用）原样；暴露 `Connection getConnection()` 与 `boolean isConnected()`。
- `TableInitializer`：`initTables` + 所有 seed INSERT 原样，依赖 `DBConnector`。
- `StorageManager`：保持**单例 + 同名同参 API**；内部委托给 DBConnector/TableInitializer；保留静态内部类 `AdminCharData/AdminItemData`（名字不变）。
  - 所有 SQL 字符串、表名、列名、`ON DUPLICATE KEY UPDATE`、`RETURN_GENERATED_KEYS`、fallback 静态数据**逐行搬运**。

## 10. 风险点提示

1. **跨包可见性**：原 `class ElementAttach`、`SkillSystem`、`NormalAttack/ElementalSkill/ElementalBurst` 是**包级**类（无 `public`），跨包后必须加 `public`，否则编译失败。重点检查。
2. **反射/枚举索引**：`ElementType.values()[rs.getInt("element")]` 依赖枚举顺序，移动枚举文件不影响顺序，但**不要重排常量**。
3. **`StorageManager.getInstance()` 被多处调用**：拆分后单例仍只应有一个 Connection；确保 DBConnector 也单例或由 StorageManager 持有唯一实例。
4. **中文文案/HTML 标签**：BattlePanel、CombatSystem 大量 `<font color=...>`、emoji（🎉☠️）原样保留，`-encoding UTF-8` 必须保留。
5. **MySQL 路径硬编码 `E:\SQL\...`**：保留，这是作者本机约定。
6. **`.class` 旧产物**：原 `bin\` 里有按"默认包类名"生成的 `.class`，分包后会冲突。**必须清空 `bin\` 再编译**。
