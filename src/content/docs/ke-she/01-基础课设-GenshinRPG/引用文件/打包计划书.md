## 1. 核心问题：硬编码的 MySQL 路径

当前 `DBConnector.java` 硬编码：
```java
File mysqlExe = new File("E:\\SQL\\mysql\\bin\\mysqld.exe");
File myIni    = new File("E:\\SQL\\mysql\\my.ini");
pb.directory(new File("E:\\SQL"));
```
别人电脑没有 `E:\SQL`，程序连不上库就 `System.exit(1)` 退出。**必须改成相对路径**，让 MySQL 跟着项目走。

## 2. MySQL 瘦身清单（关键，决定包体积）

实测 `E:\SQL\mysql` 共 **1.1 GB**，绝大部分可删。运行时只保留：

### ✅ 必须保留
| 路径 | 体积 | 用途 |
|---|---|---|
| `bin/mysqld.exe` | 49M | 数据库服务主程序（唯一必需的可执行） |
| `bin/*.dll`（与 mysqld 同目录的运行时 DLL） | ~5M | VC++ 运行时，通常已在系统，但保险保留 |
| `share/` | 9.5M | 字符集、错误信息、时区表（mysqld 启动必需） |
| `lib/libmysql.dll` | 6.9M | 客户端库（连接器用，保留保险） |
| `lib/plugin/` | — | 存储引擎插件（InnoDB 等，**mysqld 启动会加载**） |
| `my.ini` | 145B | 配置文件（路径要改成相对/自适应） |
| `LICENSE` `README` | <1M | GPL 协议，**必须带**（GPL 合规） |

### ❌ 可以删除（瘦身）
| 路径 | 体积 | 原因 |
|---|---|---|
| `bin/mysqld.pdb` | **333M** | 调试符号，运行完全不需要 |
| `bin/libprotobuf-debug.pdb` 等 `*.pdb` | ~30M | 全部 pdb 调试符号 |
| `bin/mysql.exe mysqladmin.exe mysqldump.exe mysqlcheck.exe mysqlimport.exe mysqlshow.exe mysqlslap.exe mysqlpump.exe mysqlbinlog.exe mysql_migrate_keyring.exe mysql_upgrade.exe mysql_secure_installation.exe perror.exe myisam*.exe` | 各 6-7M | 客户端/维护工具，游戏运行不需要，保留 mysqld.exe 一个即可 |
| `lib/*.lib`（libmysql.lib/mysqlclient.lib/mysqlservices.lib） | **53M** | 开发链接库，运行不需要 |
| `lib/private/` | — | 私有头，不需要 |
| `lib/mecab/` | — | 日语分词（可选删，体积小可留） |
| `include/` | 412K | C 开发头文件，不需要 |
| `docs/` | 10K | 文档，不需要 |
| `data/binlog.*` | <1M | 二进制日志，初始化时会重生 |
| `data/*.err *.pid auto.cnf` | 小 | 运行时生成 |

**data 目录单独处理**（见第 3 节）。

瘦身后预估：`mysqld.exe(49) + share(9.5) + lib(~7) + bin/dll(~5) ≈ 70M`，zip 压缩后约 **30M**。

## 3. data 目录策略（最容易出错，重点）

mysqld 需要 `datadir` 指向一个**已初始化**的数据目录（含 mysql 系统库、表结构）。两个方案：

### 方案 A（推荐）：打包干净的 data 目录
1. 把现有 `E:\SQL\mysql\data` 整个复制过来，但**清理运行时产物**：删 `binlog.*`、`*.err`、`*.pid`、`auto.cnf`、`ib_logfile*`、`ibdata1`、`undo_*`、`#innodb_temp`、`mysql.sock`。
2. **保留**：`mysql/`（系统库）、`genshin_rpg/`（游戏数据，900K）、`sys/`、`performance_schema/`、各种 `.pem`（SSL 证书，初始化生成但保留可避免重新生成失败）、`ib_buffer_pool`。
3. 风险：InnoDB 数据文件（ibdata1）被删后，启动时可能报"系统表空间不存在"。**更稳的做法是用 `mysqld --initialize` 重新生成 data**（见方案 B）。

### 方案 B（最稳）：data 目录不打包，首次启动自动初始化
1. zip 里**不带 data 目录**（或带一个空的 `data/` 占位）。
2. `start_mysql.bat` 启动前检测：若 `data\mysql` 不存在，执行：
   ```
   mysqld --defaults-file=my.ini --initialize-insecure --console
   ```
   （`--initialize-insecure` = root 空密码，与代码里密码候选 `["","root","123456"]` 匹配）。
3. 初始化后再启动 mysqld，再跑 `db\genshin_rpg.sql` 建库建表。

> **本计划采用方案 B 的混合**：打包里带一份**预先 `--initialize` 过、并已导入 genshin_rpg.sql 的干净 data**（让对方解压即有数据、零等待），同时在 `start_mysql.bat` 里加"检测 data 是否存在，否则自动初始化"的兜底。两全。

### my.ini 路径处理
原 `my.ini`：
```ini
[mysqld]
port=3306
basedir=E:\SQL\mysql
datadir=E:\SQL\mysql\data
character-set-server=utf8mb4
default-storage-engine=INNODB
secure-file-priv=""
```
问题：`basedir/datadir` 用了绝对路径，换机器就失效。**MySQL 的 `basedir/datadir` 不支持相对路径写法**（会相对 `CURRENTDIR`，行为不可靠）。

**解决：不把 basedir/datadir 写死在 my.ini，改由启动脚本动态生成 my.ini**（脚本知道自己所在目录）。详见第 5 节 `start_mysql.bat`。

## 4. 代码改造（仅 `DBConnector.java`，最小改动）

### 4.1 路径改成相对工作目录
原代码用 `E:\\SQL\\mysql\\bin\\mysqld.exe`。改成：从 **当前工作目录** 或 **jar/类路径所在目录** 推导 MySQL 路径。

采用约定：**MySQL 子目录与启动脚本同级**。即包内布局：
```
GenshinRPG/
├─ run.bat                    ← 双击启动入口
├─ start_mysql.bat            ← 启动/初始化 MySQL（被 run.bat 调用）
├─ stop_mysql.bat             ← 关闭 MySQL
├─ mysql/                     ← 瘦身后的免安装 MySQL
│   ├─ bin/mysqld.exe
│   ├─ share/
│   ├─ lib/libmysql.dll, plugin/
│   ├─ data/                  ← 干净 data（含 genshin_rpg）
│   └─ my.ini                 ← 由 start_mysql.bat 动态生成
├─ lib/
│   └─ mysql-connector-j-8.0.33.jar
├─ bin/                       ← 编译产物（可预编译好，对方免编译）
├─ src/                       ← 源码（分包重构版）
├─ db/
│   └─ genshin_rpg.sql        ← 建库脚本（首次初始化兜底用）
├─ README.txt                 ← 使用说明
└─ .gitignore
```

### 4.2 DBConnector.java 改动点（精确到行）

定位（重构版 `refactored/src/com/genshin/rpg/dao/DBConnector.java`，约 50-65 行）：

**原代码**：
```java
File mysqlExe = new File("E:\\SQL\\mysql\\bin\\mysqld.exe");
File myIni = new File("E:\\SQL\\mysql\\my.ini");
if (mysqlExe.exists() && myIni.exists()) {
    ProcessBuilder pb = new ProcessBuilder(
        mysqlExe.getAbsolutePath(),
        "--defaults-file=" + myIni.getAbsolutePath()
    );
    pb.directory(new File("E:\\SQL"));
    pb.start();
    Thread.sleep(4000);
    connected = tryConnect();
}
```

**改为**（用相对路径，基准 = 程序工作目录 = 包根目录）：
```java
// 内置 MySQL 子目录路径：相对当前工作目录
File mysqlDir  = new File("mysql");
File mysqlExe  = new File(mysqlDir, "bin/mysqld.exe");
File myIni     = new File(mysqlDir, "my.ini");
if (mysqlExe.exists() && myIni.exists()) {
    ProcessBuilder pb = new ProcessBuilder(
        mysqlExe.getAbsolutePath(),
        "--defaults-file=" + myIni.getAbsolutePath()
    );
    pb.directory(mysqlDir.getAbsoluteFile());   // 工作目录设为 mysql/，便于相对 data 路径
    pb.redirectErrorStream(true);
    pb.start();
    Thread.sleep(4000);
    connected = tryConnect();
}
```

> ⚠️ **红线**：除路径外，其余逻辑**一字不改**——密码候选 `["","root","123456"]`、连接串 `jdbc:mysql://localhost:3306/...`、库名 `genshin_rpg`、`Thread.sleep(4000)`、连不上则 `JOptionPane + System.exit(1)` 全部保留。

## 5. 启动脚本设计

### 5.1 `start_mysql.bat`（核心，处理路径自适应 + 首次初始化）
```bat
@echo off
chcp 65001 >nul
cd /d "%~dp0"

echo [MySQL] 检查数据库服务...
REM 若 3306 已通，说明 MySQL 已在跑（本机或其他），直接跳过启动
mysql\bin\mysql.exe -uroot -e "SELECT 1" >nul 2>&1
REM 注意：上面这行需要 mysql.exe，我们瘦身时删了。改用端口探测：

REM 用 powershell 探测端口（避免依赖 mysql.exe）
powershell -Command "try { $c=New-Object Net.Sockets.TcpClient; $c.Connect('localhost',3306); $c.Close(); exit 0 } catch { exit 1 }"
if %errorlevel%==0 (
    echo [MySQL] 检测到 3306 端口已开放，跳过启动。
    goto :eof
)

REM 端口未开，启动内置 MySQL
echo [MySQL] 正在启动内置免安装 MySQL...

REM 动态生成 my.ini（路径自适应本机）
set "BASE=%cd%\mysql"
set "DATA=%BASE%\data"
(
    echo [mysqld]
    echo port=3306
    echo basedir=%BASE%
    echo datadir=%DATA%
    echo character-set-server=utf8mb4
    echo default-storage-engine=INNODB
    echo secure-file-priv=""
    echo skip-grant-tables
) > mysql\my.ini

REM 若 data 未初始化，先初始化（首次运行）
if not exist "mysql\data\mysql" (
    echo [MySQL] 首次运行，正在初始化数据目录（约 10 秒）...
    mysql\bin\mysqld.exe --defaults-file=mysql\my.ini --initialize-insecure --console
)

REM 后台启动 mysqld
start "" /B mysql\bin\mysqld.exe --defaults-file=mysql\my.ini

REM 等待启动（最多 30 秒，每秒探测一次端口）
set /a wait=0
:waitloop
powershell -Command "try { $c=New-Object Net.Sockets.TcpClient; $c.Connect('localhost',3306); $c.Close(); exit 0 } catch { exit 1 }"
if %errorlevel%==0 goto :ready
set /a wait+=1
if %wait% geq 30 goto :timeout
timeout /t 1 /nobreak >nul
goto :waitloop

:ready
echo [MySQL] 启动成功。
goto :eof

:timeout
echo [MySQL] 启动超时！请检查 mysql\data 目录或 3306 端口占用。
exit /b 1
```

> 关键点：
> - 用 `skip-grant-tables` 让 root 免密（与代码密码候选 `""` 匹配）。
> - `--initialize-insecure` 首次初始化（root 空密码）。
> - 端口探测替代依赖 mysql.exe（瘦身已删）。
> - 用 PowerShell 探测端口兼容性好（Win7+/PS 自带）。

### 5.2 `run.bat`（用户双击入口）
```bat
@echo off
chcp 65001 >nul
cd /d "%~dp0"

echo ========================================
echo   原神文字战斗模拟系统 - 启动中
echo ========================================

REM 1. 启动内置 MySQL
call start_mysql.bat
if %errorlevel% neq 0 (
    echo [错误] MySQL 启动失败，程序无法继续。
    pause
    exit /b 1
)

REM 2. 启动 Java 程序
echo [应用] 正在启动...
set "CP=bin;lib\mysql-connector-j-8.0.33.jar"
java -cp "%CP%" com.genshin.rpg.Main

REM 3. 退出时关闭 MySQL（可选，避免残留进程）
echo.
echo [清理] 正在关闭数据库服务...
call stop_mysql.bat
pause
```

### 5.3 `stop_mysql.bat`
```bat
@echo off
cd /d "%~dp0"
REM 用 mysqladmin 关闭需要 mysqladmin.exe（已删）。改用任务管理方式：
REM 方案：存储 mysqld 的 PID 启动时记录，关闭时按 PID 杀进程。
REM 简化版：直接按进程名结束本目录启动的 mysqld（不影响系统服务）
REM 安全做法：记录 PID
if exist mysql\mysqld.pid (
    for /f %%P in (mysql\mysqld.pid) do taskkill /PID %%P /F >nul 2>&1
    del mysql\mysqld.pid
) else (
    REM 兜底：提示用户手动关
    echo [提示] 未找到 PID 记录。如需关闭，可在任务管理器结束 mysqld.exe。
)
```
（启动脚本里 `start "" /B mysqld` 改为先拿 PID：`start "" /B cmd /c "mysqld ... & echo !PID! > mysql\mysqld.pid"` 较复杂，执行 AI 可简化为"程序退出后留 MySQL 运行，下次启动更快"，在 README 里说明。）

## 9. 风险与对策

| 风险 | 对策 |
|---|---|
| mysqld.exe 缺少 `lib/plugin` 无法启动 | 保留 `lib\plugin\`（InnoDB 等引擎插件） |
| `--initialize-insecure` 在某些 MySQL 版本行为差异 | 打包已初始化的 data 兜底（方案 B 混合） |
| 对方机器已有 MySQL 占 3306 | start_mysql.bat 端口探测，已开则复用 |
| 路径含中文/空格导致 mysqld 报错 | README 提示解压到无中文空格路径 |
| GPL 协议（MySQL 是 GPL） | 保留 LICENSE/README，分发说明注明"内含 MySQL Community Server (GPL)" |
| binlog 删除后启动失败 | 用 `--initialize` 重生而非手动删 |
