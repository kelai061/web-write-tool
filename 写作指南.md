## 1. 三条必须记死的规则（违反就报错或 404）

### 规则 ① URL 里没有 `.md`，没有 `docs/`

| 文件路径                                  | 对应的 URL            |
| ------------------------------------- | ------------------ |
| `src\content\docs\index.mdx`          | `/`                |
| `src\content\docs\about.mdx`          | `/about/`          |
| `src\content\docs\guide\lesson-1.mdx` | `/guide/lesson-1/` |

链接永远写**末尾带斜杠**的形式：`[链接](/guide/lesson-1/)` ✅
不要写 `/docs/guide/lesson-1/` ❌、不要写 `lesson-1.md` ❌、不要写 `lesson-1.html` ❌。

### 规则 ② `.md` vs `.mdx`

- **纯文字 + 简单 Markdown**（标题、列表、代码块、图片、表格）→ 用 `.md`
- **需要 `import` 组件或图片**（Card、Steps、Tabs、Aside、import 截图）→ 用 `.mdx`

> 不确定就用 `.mdx`，永远不会错。模板里默认用 `.mdx`。

### 规则 ③ ⚠️ MDX 大坑：JSX 组件里不能写裸代码块或 `{大括号}`

**错**（会报错或静默失败）：

```mdx
<Card title="示例">
看这段代码：
```js
const x = { a: 1 };   ← 大括号会被当 JSX 表达式
```

</Card>
```

**对**：把代码示例放在 `<Card>` 外面，或者把 `{` 转义成 `\{`：

```mdx
<Card title="示例">
  这张卡片只放说明文字。
</Card>

代码示例放在卡片外面：

​```js
const x = { a: 1 };
​```
```

> **任何在 `<组件>...</组件>` 标签之间的内容，都不要写代码块或裸 `{`。**

## 3. 写一篇文章（最常见操作）

### 步骤

1. 决定文章放哪。假设放在 `guide\` 类目下，叫 `my-article`。
2. 新建文件 `E:\AI\zcode\object\Web imitation\tool\src\content\docs\guide\my-article.mdx`
3. 写入下面模板（复制粘贴改内容）：

```mdx
---
title: 我的文章标题
description: 一句话描述，会进搜索结果和 SEO（建议 50-100 字）
sidebar:
  order: 3   # 在侧边栏里的位置，数字越小越靠上；不写就按文件名排
---

# 我的文章标题

正文从这里开始。直接写 Markdown。

## 第一节

普通段落。**加粗**、*斜体*、`行内代码`、[链接](https://example.com)。

- 列表项一
- 列表项二

### 子节

代码块（自动高亮 + 复制按钮）：

​```js
function hello(name) {
  return `Hello, ${name}!`;
}
​```

> [!NOTE]
> 这是一个彩色提示框，type 可以是 NOTE / TIP / WARNING / CAUTION。
```

4. 保存。浏览器（dev 还开着的话）自动刷新就能看到。
5. **如果要进侧边栏**：只要文件在 `astro.config.mjs` 里某个 `autogenerate.directory`
   指向的文件夹下，会自动出现，**不用改任何配置**。

### frontmatter 字段速查

| 字段                 | 作用                    | 示例                                                  |
| ------------------ | --------------------- | --------------------------------------------------- |
| `title`            | 页面标题（必填）              | `title: 安装教程`                                       |
| `description`      | SEO + 搜索描述            | `description: 5 分钟装好环境`                             |
| `sidebar.order`    | 侧边栏排序                 | `sidebar: { order: 2 }`                             |
| `sidebar.label`    | 侧边栏里显示的名字（不写就用 title） | `sidebar: { label: '第一章' }`                         |
| `sidebar.badge`    | 侧边栏条目旁的徽章             | `sidebar: { badge: { text: '新', variant: 'tip' } }` |
| `draft: true`      | 草稿，不发布                | `draft: true`                                       |
| `template: splash` | 用首页大图模板（只首页用）         | 见 `index.mdx`                                       |

## 4. 写一个系列教程 / 一个类目

### 场景：你要写一个《Python 入门》系列，10 篇文章

**步骤 1**：建文件夹

```
src\content\docs\python\          ← 新建这个文件夹
```

**步骤 2**：在 `astro.config.mjs` 的 `sidebar` 数组里加一条分组：

```js
sidebar: [
  // ... 已有的分组 ...
  {
    label: 'Python 入门',                    // 侧边栏显示的分组名
    autogenerate: { directory: 'python' },   // 文件夹名（对应 src\content\docs\python\）
  },
],
```

**步骤 3**：在 `python\` 下建 10 个文件，用 `order` 排序：

```
python\
├─ 01-intro.mdx        ← frontmatter: sidebar: { order: 1 }
├─ 02-variables.mdx    ← frontmatter: sidebar: { order: 2 }
├─ 03-control-flow.mdx ← frontmatter: sidebar: { order: 3 }
├─ ...
└─ 10-project.mdx      ← frontmatter: sidebar: { order: 10 }
```

文件名建议用 `01-`、`02-` 前缀，这样文件管理器里也按顺序排；
但**侧边栏顺序由 `sidebar.order` 决定**，不是文件名。

**步骤 4**：文章之间互相链接：

```mdx
[上一课：变量](/python/02-variables/)
[下一课：控制流](/python/03-control-flow/)
```

> URL 是文件夹 + 文件名（去掉 `.mdx`），末尾斜杠。

**步骤 5**：保存 → dev 服务器自动刷新 → 侧边栏多出 "Python 入门" 分组。

## 5. 想精确控制侧边栏顺序 / 想隐藏某些文章

如果 `autogenerate` 不够灵活（比如想跳号、想显示别名、想临时下线一篇），
改用手动 `items`：

```js
sidebar: [
  {
    label: 'Python 入门',
    // 手动列出每条，顺序就是数组顺序
    items: [
      { label: '开篇',        link: '/python/01-intro/' },
      { label: '变量与类型',   link: '/python/02-variables/' },
      { label: '控制流',       link: '/python/03-control-flow/' },
      // 不列出来的文章不会出现在侧边栏（但仍可通过 URL 访问）
    ],
  },
]
```

> 一篇文章**没在侧边栏里**，但 URL 能打开 → 把它当"隐藏页面"用（比如下载页、致谢页）。

## 7. 常用 Markdown / 组件速查（直接抄）

### 标题（自动进右侧目录）

```md
# 一级标题（每篇只有一个，通常用 frontmatter 的 title 代替）
## 二级标题
### 三级标题
```

### 代码块（自动高亮 + 复制按钮 + 文件名 + 行高亮）

```md
​```js title="hello.js" {2}
function hello(name) {
  return `Hello, ${name}!`;   ← 这一行被高亮
}
​```
```

支持的语言：js / ts / python / bash / json / yaml / html / css / sql / go / rust / java / …（100+）

### 彩色提示框（GitHub aside 语法，最简单）

```md
> [!NOTE]
> 普通提示（蓝色）

> [!TIP]
> 小技巧（绿色）

> [!WARNING]
> 警告（黄色）

> [!CAUTION]
> 危险操作（红色）
```

### 表格

```md
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| a   | b   | c   |
| d   | e   | f   |
```

### 图片

**两种方式**：

方式 A（推荐，会被自动压缩优化）：把图片放 `src\assets\`，文章用 `.mdx`：

```mdx
import cover from '../../assets/cover.png';

![封面图](cover)
```

方式 B（简单）：把图片放 `public\` 下，用绝对路径（图片不会被优化）：

```md
![封面图](/cover.png)
```

> `public\cover.png` → URL 是 `/cover.png`

### 卡片网格（首页/导航页常用）

```mdx
import { CardGrid, Card } from '@astrojs/starlight/components';

<CardGrid stagger>
  <Card title="入门" icon="seti:default" href="/getting-started/intro/">
    简短描述
  </Card>
  <Card title="进阶" icon="seti:default" href="/guide/lesson-1/">
    简短描述
  </Card>
</CardGrid>
```

图标可选值见 https://github.com/troybetz/react-seti 或用 Starlight 内置图标名。

### 步骤（带连接线的有序列表）

```mdx
import { Steps } from '@astrojs/starlight/components';

<Steps>
  1. **第一步**

     说明文字（缩进 4 空格）。

  2. **第二步**

     说明文字。
</Steps>
```

> ⚠️ Steps 内部必须用 `1. 2. 3.` 有序列表，**不要**用 `###` 标题。

### 选项卡（同主题不同语言/工具）

```mdx
import { Tabs, TabItem } from '@astrojs/starlight/components';

<Tabs>

## 8. 排错清单（构建失败 / 显示不对时按顺序查）

| 症状                                          | 排查                                                                                                                                                                                                      |
| ------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Build failed` 报 "xxx is not exported"      | 你 `import` 了 Starlight 不存在的组件。当前可用：`Card, CardGrid, Steps, Tabs, TabItem, Aside, Badge, Icon, LinkCard, LinkButton, FileTree, Code`。**没有 `Note/Tip/Warning/Caution`**（用 `> [!NOTE]` 语法或 `<Aside>` 组件代替） |
| `<Steps>` 报 "expects a single ordered list" | Steps 里必须用 `1. 2. 3.`，不是 `###` 标题                                                                                                                                                                       |
| 构建成功但只有 1 页                                 | 99% 是某个 `.mdx` 文件里 JSX 组件（如 `<Card>`）内部有裸 ```` ``` ```` 代码块或 `{` 大括号，导致 MDX 解析吞掉了后续所有内容。把这些移到组件外                                                                                                        |
| 点链接 404                                     | URL 写错了。规则见第 1 节。检查：没有 `/docs/` 前缀、没有 `.md` 后缀、末尾有 `/`                                                                                                                                                  |
| 侧边栏没出现新文章                                   | 文件不在 `astro.config.mjs` 里某个 `autogenerate.directory` 指向的文件夹下；或文件顶部 frontmatter 缺了 `---`                                                                                                                 |
| 中文 URL 变 `%E4%B8%AD`                        | 文件名用了中文。建议文件名用英文/拼音（URL 干净），`title` 用中文                                                                                                                                                                 |
| 改了 config 没生效                               | dev 服务器需要 Ctrl+C 重启（`astro.config.mjs` 改动不会热刷新）                                                                                                                                                         |

## 9. 部署上线（写完想发到互联网）

### 方式 A：Vercel（推荐，免费，全球 CDN）

1. 把 `tool\` 整个目录传到 GitHub（建个新仓库）
   - 注意 `.gitignore` 已经排除了 `node_modules\` 和 `dist\`
2. 去 https://vercel.com 注册（用 GitHub 账号一键登录）
3. 点 "New Project" → 选你的仓库 → Vercel 自动识别 Astro → 直接 Deploy
4. 等 1 分钟，拿到 `xxx.vercel.app` 的免费域名，可绑定自定义域名
5. 以后只要 `git push`，Vercel 自动重新构建发布

**Vercel 关键设置**（一般会自动填对，不对就手动改）：

- Framework Preset: `Astro`
- Build Command: `npm run build`
- Output Directory: `dist`

### 方式 B：Cloudflare Pages（同样免费）

1. GitHub 建仓库
2. https://pages.cloudflare.com → 连接 GitHub
3. 构建设置：
   - Framework preset: `Astro`
   - Build command: `npm run build`
   - Build output directory: `dist`
4. 保存 → 自动部署

### 方式 C：纯静态托管（任何能放 HTML 的地方）

```bash
npm run build
```

然后把 `dist\` 文件夹里的所有文件传到：

- GitHub Pages、Netlify、阿里云 OSS、腾讯云 COS、七牛云、自己的 Nginx 服务器…

## 11. 给执行 AI 的红线

1. **永远不要在 `<组件>...</组件>` 之间写代码块或裸 `{`**。代码示例一律放在组件外面。
2. **永远不要 `import` Starlight 不导出的组件**。可用清单见第 8 节。
3. **链接永远带末尾斜杠，永远不要 `/docs/` 前缀，永远不要 `.md` 后缀**。
4. **改完 `astro.config.mjs` 必须重启 dev 服务器**（Ctrl+C 后重新 `npm run dev`）。
5. **文件名用英文/拼音，`title` 才用中文**，避免 URL 乱码。
6. **每次新增/修改文章后，跑一次 `npm run build` 验证**，确认 "X page(s) built" 数字正确。
